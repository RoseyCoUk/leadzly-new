## Spacing Scale

All spacing uses CSS custom properties declared in `style.css :root`. Executor MUST use tokens, not hardcoded values.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| `--space-2` | 8px | Gap between inline badge items in trust bar |
| `--space-3` | 12px | Label margin-bottom (section-label pattern) |
| `--space-4` | 16px | Default element spacing, card padding inner |
| `--space-6` | 24px | Card padding, grid gap |
| `--space-8` | 32px | Trust bar padding-inline on desktop |
| `--space-10` | 40px | Section header margin-bottom (min) |
| `--space-12` | 48px | Section padding (min via clamp) |
| `--space-16` | 64px | Section padding (max via clamp), section header margin-bottom (max) |
| `--space-24` | 96px | Section padding (max on large viewport via clamp) |

**Pattern for section padding (all existing sections):**
`padding: clamp(var(--space-12), 8vw, var(--space-24)) 0;`
All new sections in this phase MUST follow this exact clamp pattern.

Exceptions: Logo strip images in trust bar — minimum height 32px, maximum height 48px via `max-height: 48px`. Not an 8-point exception; logos use intrinsic sizing constrained by max-height.

**Trust bar background decision (D-11, Claude's discretion):**
Use `var(--section-tint)` (#f8fafb). The hero above uses white and the pain points section below uses a green tint. A light grey trust bar creates a clean visual break without competing with either adjacent section.

**Accent reserved for (explicit list):**
- `.section-label` text colour on all section headers
- Industry tag chip text + background tint
- "100+ meetings/month" stat badge text and border
- "30-Day Guarantee" badge icon/checkmark fill
- Focus ring (already defined globally in `base.css` — do not override)

Accent is NOT used for: general body text, card borders, image borders, background fills of full sections.

## Component Inventory

New CSS classes to be authored in `page.css` for this phase. All existing shared classes (`.section-header`, `.section-label`, `.section-title`, `.section-subtitle`, `.container`, `.btn`, `.btn--primary`, `.fade-in`) are reused without modification.

### Trust Bar — `.trust-bar`

```
.trust-bar
  .trust-bar__inner         — flex row, centered, gap var(--space-8), wraps on mobile
    .trust-bar__logos       — flex row of greyscale logo images
      img                   — filter: grayscale(100%), opacity: 0.6, max-height: 40px
    .trust-bar__divider     — 1px vertical line, color-border, hidden on mobile
    .trust-bar__badge       — inline flex, gap --space-2, --text-sm weight 600
      .trust-bar__badge--stat    — "100+ meetings/month"
      .trust-bar__badge--guarantee — "30-Day Guarantee"
```

- Background: `var(--section-tint)`
- Padding: `var(--space-4) 0` (slim band — not a full content section)
- Logo row gap: `var(--space-6)`
- Logos: `filter: grayscale(100%)`, `opacity: 0.6`, `max-height: 40px`, `width: auto`
- On hover (logos): `opacity: 1`, `filter: grayscale(0%)`, `transition: var(--transition-interactive)` — subtle brand reveal
- Badges: green text `var(--color-primary)`, weight 600, font-size `--text-sm`
- Dividers between sections: visible `var(--color-divider)` 1px vertical, `display: none` below 640px

### Proven Across Industries Section — `.proven` (replaces `.case-studies`)

```
.proven
  .container
    .section-header
      .section-label        — "Results"
      .section-title        — "Proven Across Industries"
      .section-subtitle     — "100+ meetings booked monthly across the Elevateo portfolio"
    .proven__grid           — 2×2 grid on desktop, 1-column on mobile
      .proven__card (×4)
        .proven__card-tag   — industry label chip
        .proven__card-outcome — one-line result description
        .proven__card-detail  — supporting detail line
```

- Section background: `var(--section-white)` (D-06)
- Grid: `grid-template-columns: repeat(2, 1fr)` at ≥640px, `1fr` below (D-04 — 2×2 grid, Claude's discretion confirmed)
- Grid gap: `var(--space-6)`
- Card padding: `var(--space-6)`
- Card background: `var(--color-surface)` (#f9fafb)
- Card border: `1px solid var(--color-border)`
- Card border-radius: `var(--radius-lg)`
- Card shadow: `var(--shadow-sm)`
- Card already receives `.fade-in` class (Phase 6 wires the animation JS)
- Industry tag chip: background `var(--color-primary-soft)`, color `var(--color-primary)`, font-size `--text-xs`, weight 600, border-radius `var(--radius-full)`, padding `var(--space-1) var(--space-3)`
- Outcome line: font-size `--text-lg`, weight 700, color `var(--color-text)`, margin-top `var(--space-3)`
- Detail line: font-size `--text-sm`, weight 400, color `var(--color-text-muted)`, line-height 1.6, margin-top `var(--space-2)`

**4 cards — content specification:**

| Card | Industry Tag | Outcome Line | Detail Line |
|------|-------------|-------------|------------|
| 1 | Digital Marketing | Consistent pipeline for a growing agency | Outbound calling + email sequences targeting UK SME decision-makers |
| 2 | AI & Automation | Early-stage SaaS demos booked at scale | LinkedIn prospecting + follow-up cadences for a pre-revenue product |
| 3 | SaaS & Software | Qualified discovery calls without an internal SDR | Multi-channel outreach across cold email and telesales |
| 4 | E-Commerce | New B2B retail partnerships opened monthly | Targeted cold outreach to buyers at independent and chain retailers |

  <action>
Create a new file `sitemap.xml` at the project root (same directory as index.html) with exactly this content:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://leadzly.co/</loc>
    <lastmod>2026-04-28</lastmod>
  </url>
</urlset>
```

Line-by-line requirements:
- Line 1: XML declaration — `<?xml version="1.0" encoding="UTF-8"?>`
- Line 2: `<urlset>` opening tag with the exact sitemaps.org 0.9 namespace: `xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"`
- Line 3: `<url>` opening tag (2-space indent)
- Line 4: `<loc>https://leadzly.co/</loc>` — WITH trailing slash (matches canonical href from Plan 01; must be consistent)
- Line 5: `<lastmod>2026-04-28</lastmod>` — W3C Datetime YYYY-MM-DD format; use 2026-04-28 as the last-modified date
- Line 6: `</url>` closing tag
- Line 7: `</urlset>` closing tag

Do NOT include a `<changefreq>` or `<priority>` tag — they are optional and Google ignores them. Do NOT use a sitemapindex wrapper (only needed for multiple sitemaps). File must be UTF-8, Unix line endings (LF).
  </action>
  <verify>
    <automated>grep 'leadzly.co' sitemap.xml && grep 'lastmod' sitemap.xml</automated>
  </verify>

    - sitemap.xml contains: `xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"`
    - sitemap.xml contains: `<loc>https://leadzly.co/</loc>` (with trailing slash — no trailing-slash mismatch with canonical)

  </acceptance_criteria>
